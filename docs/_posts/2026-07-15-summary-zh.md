## [Claude web_fetch 漏洞导致数据窃取攻击](https://simonwillison.net/2026/Jul/15/claude-web-fetch-exfiltration/#atom-everything) ⭐️ 8.0/10

安全研究员 Ayush Paul 发现了 Anthropic 的 Claude web_fetch 工具中的一个漏洞，攻击者通过诱骗 AI 遵循恶意蜜罐页面中嵌入的 URL 链，成功窃取了用户的私人数据（姓名、所在城市和雇主名称）。 此次攻击展示了一个关键的 AI 安全风险：即使是像 web_fetch 这样精心设计的工具，也可能通过提示注入被绕过，威胁用户隐私，凸显了在 AI 代理中需要更强健的隔离机制。 该攻击利用了一个设计缺陷：web_fetch 被允许跟随所获取页面中返回的链接，这违反了 Anthropic 规定的“仅导航到用户或 web_search 提供的精确 URL”的规则。Anthropic 声称已内部发现此问题并修复了该漏洞，移除了该能力，且未支付漏洞赏金。

rss · Simon Willison · 7月15日 14:21

**背景**: “致命三要素”是一个已知的 AI 安全概念，包含三个条件：访问私人数据、暴露于不受信任的内容（例如通过 web 获取）以及数据窃取机制（例如通过 URL）。Claude 的 web_fetch 工具旨在获取特定 URL 的内容，但此次攻击表明提示注入如何绕过这些防护措施。

## [T4 上 PyTorch 模型比 A100 慢 170 倍：可能原因探讨](https://www.reddit.com/r/MachineLearning/comments/1ux6a9x/pytorch_model_running_170x_slower_on_t4_vs_a100/) ⭐️ 8.0/10

一位用户报告，在纯 FP32 模式下，其点追踪 PyTorch 模型在 NVIDIA T4 GPU 上比 A100 慢约 170 倍，且两个 GPU 利用率均达 99%。 这一极端性能差距凸显了 GPU 架构差异（如 Tensor Core、内存带宽）对 4D 相关体积和 Transformer 等计算密集型操作的影响，进而影响模型部署决策。 该模型构建用于帧间密集匹配的局部 4D 相关体积，并采用 Transformer 层，全部使用 FP32 精度；T4 缺乏 FP32 的 Tensor Core，且内存带宽远低于 A100，这很可能是瓶颈所在。

reddit · r/MachineLearning · /u/Future-Structure-296 · 7月15日 13:44

**背景**: NVIDIA T4 与 A100 GPU 差异显著：A100 具备混合精度 Tensor Core（但 FP32 不支持），更高内存带宽（1555 GB/s 对比 320 GB/s）及更大缓存。4D 相关体积常见于 RAFT 等光流模型，需计算所有像素特征对之间的点积，属于内存密集型操作。T4 缺乏 FP32 Tensor Core，只能依赖 CUDA 核心，此类操作速度较慢。

## [Telegram 推出无服务器机器人平台](https://core.telegram.org/bots/serverless) ⭐️ 8.0/10

Telegram 正式推出无服务器平台，开发者无需管理服务器，可直接在 Telegram 基础设施上运行机器人和 Mini App 的 JavaScript 后端代码。 这简化了机器人的部署和扩展，降低了开发者的运维负担和创建 Telegram 机器人的门槛。通过在消息平台内提供第一方无服务器计算，增强了 Telegram 的生态系统。 该平台使用与 Bot API 临近的隔离 V8 沙箱环境，内置 SQLite 数据库，并通过一条命令 'npx tgcloud push' 即可完成部署。仅支持普通 JavaScript 模块。

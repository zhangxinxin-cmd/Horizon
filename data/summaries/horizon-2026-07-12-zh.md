## [Claude Code 相同任务消耗 33k tokens，OpenCode 仅 7k](https://systima.ai/blog/claude-code-vs-opencode-token-overhead) ⭐️ 8.0/10

一项研究发现，Claude Code 在读取用户提示前发送约 33,000 个 tokens，而 OpenCode 在相同任务中仅发送约 7,000 个 tokens，表明 Claude Code 存在显著的 token 额外开销。 这种 token 低效直接增加了使用 Claude Code 的开发者的成本，尤其是在大型或迭代编码任务中，并凸显了 harness 设计和缓存策略如何影响 AI 编码工具的经济性。 额外开销源于 Claude Code 低效的提示缓存和更大的系统提示 harness，而 OpenCode 采用了更优化的方案。该研究记录了工具与 Anthropic 端点之间的所有请求以捕获使用情况。

hackernews · systima · 7月12日 18:25 · [社区讨论](https://news.ycombinator.com/item?id=48883275)

**背景**: Token 额外开销是指 AI 编码工具在处理用户实际输入之前，由其系统提示、工具定义和缓存机制消耗的额外 tokens。提示缓存是一种重用计算状态的技术，但如果未优化，会浪费 tokens。Claude Code 和 OpenCode 等智能编码工具充当用户与大语言模型之间的 harness，harness 的设计对 token 使用有很大影响。

## [带状疱疹疫苗或可降低痴呆风险](https://www.economist.com/leaders/2026/07/09/a-no-brainer-for-protecting-your-brain) ⭐️ 8.0/10

一项采用断点回归设计的英国研究发现，有资格接种带状疱疹疫苗的人在七年内被诊断出痴呆的概率更低。 如果该效应是因果性的，这可能提供一种安全且廉价的方法来降低痴呆风险，惠及全球数百万人。 该研究利用了英国疫苗接种资格严格的年龄截止点，比较了出生日期恰好在该截止点前后的人群，以估计因果效应。

hackernews · saikatsg · 7月12日 15:23 · [社区讨论](https://news.ycombinator.com/item?id=48881874)

**背景**: 断点回归设计 (RDD) 是一种准实验方法，用于在无法随机分组时估计因果效应。它比较阈值（如年龄截止点）两侧受试者的结果，推断治疗影响。RDD 在健康和社会科学中广泛用于评估政策干预。

## [Ghostel.el：基于 libghostty 的快速 Emacs 终端模拟器](https://dakra.github.io/ghostel/) ⭐️ 8.0/10

Ghostel.el 是一个基于 libghostty-vt 构建的新 Emacs 终端模拟器，相比 vterm 和 eat 等现有方案，性能大幅提升。它结合了用于渲染和 PTY I/O 的原生 Zig 模块以及用于键映射、缓冲区和远程进程集成的 Elisp。 该项目解决了 Emacs 终端模拟器长期存在的性能问题，使 TUI 应用程序运行更流畅，输入处理更可靠。它为 Emacs 终端集成树立了新标准，有望成为需要快速嵌入式终端的用户的首选。 Ghostel 由 libghostty-vt 驱动——与独立 Ghostty 终端相同的 VT 引擎，并以 Zig 编写的原生动态模块实现。它具有多种输入模式，以解决终端按键消耗与编辑器命令处理之间的冲突。

hackernews · signa11 · 7月12日 08:52 · [社区讨论](https://news.ycombinator.com/item?id=48879504)

**背景**: Emacs 是一个高度可扩展的文本编辑器，可通过 vterm 和 eat 等包在内部运行终端。然而，这些终端经常出现性能问题，尤其是在图形化 TUI 应用程序中。libghostty 是 Mitchell Hashimoto 开发的一个可嵌入库，提供快速、现代的终端模拟器引擎。Ghostel.el 利用该引擎为 Emacs 带来高性能终端仿真。

## [xAI Grok CLI 默认上传整个代码库及密钥文件](https://gist.github.com/cereblab/dc9a40bc26120f4540e4e09b75ffb547) ⭐️ 8.0/10

安全研究人员发现，xAI 的 Grok Build CLI（版本 0.2.93）会默认将整个代码库及 .env 等敏感文件通过 Google Cloud Storage 上传至 xAI 服务器，且关闭“改进模型”开关也无法阻止上传。 这对使用 Grok Build 的开发者构成严重的隐私和安全风险，可能导致专有代码和凭证泄露。同时也引发对 xAI 工具的信任问题，可能影响其采用。 该 CLI 将文件内容嵌入聊天请求，同时将整个 git 仓库以 git bundle 形式上传，即使提示词要求不要读取某个文件。在 12 GB 仓库的测试中，成功上传超过 5 GiB 数据且未遭服务器拒绝。

telegram · zaihuapd · 7月12日 04:19

**背景**: Grok Build 是 xAI 推出的 AI 编程 CLI 工具，使用 Grok 模型辅助编程工作。git bundle 是一种将 Git 对象和引用打包为单个文件的格式，常用于离线传输。该安全分析通过抓包进行，仅确认了数据传输和存储行为，未证明 xAI 使用这些数据进行模型训练。
